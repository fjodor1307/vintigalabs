## 2026-05-05 — Page Template restructure + responsive rail

- DS nav: pulled `Page Template` out of `Data Display`. New `Templates` sub-group under Components — first entry `Products` (id `ds-template-products`). Leaves room for `Clubs` and other named templates.
- `PageTemplate` is now responsive — rail used to be `hidden lg:flex` and disappeared below 1024 px. Now it stacks below the main column on smaller widths (full-width, `px-vintiga-xl pb-vintiga-xl`) and switches back to a 360-px right column at `lg+`.
- Products prototype refactored to follow the canonical pattern: plain string title (uses `typo-title-screen` from `PageTemplate`), `Save` + kebab `IconButton` actions (Duplicate / Archive), and `RailSection`s passed directly to `PageTemplate`'s `rail` slot. Drops the custom thumbnail/tags header and the `<RightRail>` wrapper that was double-padding the aside.
- Clubs prototype rail aligned the same way — `ClubDetailsRail` no longer wraps in `<RightRail>`. Both editors now share identical rail spacing.

## 2026-05-04 — KpiCard: three variants from Figma (132:6062)

Brought `@ds/shared/KpiCard` in line with the Figma source of truth. The component now covers all three documented variants on a single API:

1. **Plain** — `label` + `value` + `icon`
2. **Goal / progress** — adds `goalLabel` + `progressPercent` (renders the progress strip below the value)
3. **Status pill** — adds `status?: ReactNode` slot (e.g. `<Tag variant="outline">Awaiting first sale</Tag>`) inline with the value. Combine with goal/progress freely — the "AOV $0 / Awaiting first order / No goal set 0%" empty state from Figma renders correctly.

Other tweaks:
- Icon container resized 40 → **36 px** (matches Figma) and forces 20 px child SVGs via `[&>svg]:w-5 [&>svg]:h-5`. Existing usages don't need to pass `w-* h-*` on the icon.
- Progress section now renders whenever `goalLabel` is set, even without an explicit `progressPercent` (defaults to 0% — supports the "No goal set" state).

Style guide entry refreshed at `#/web/design-system → Data Display → KPI Card` with a row per variant.

## 2026-05-05 — `PageTemplate` DS layout primitive (Figma 5640:28214)

New shared component `@ds/shared/PageTemplate` — the canonical editor / detail page layout. Composes existing DS pieces (Breadcrumb, Button, IconButton, SegmentedControl, SectionCard, RailSection) into the standard shape:

```
┌──────────────────────────────────────────┬─────────────┐
│ Home > Section > Page                    │             │
│                                          │  Title      │
│ Title                  [Save] [⋯]        │  ┌────────┐ │
│                                          │  │ rail   │ │
│ [Tabs · · · · · ]                        │  │ card   │ │
│                                          │  └────────┘ │
│ ┌────────────────────────────────────┐   │             │
│ │ SectionCard …                       │   │             │
│ └────────────────────────────────────┘   │             │
└──────────────────────────────────────────┴─────────────┘
```

- Main column: `p-vintiga-xl` (32 px) with 32-px gap between header rows (breadcrumb / title / tabs / body).
- Body sections gap: configurable, default `lg` (24 px).
- Right rail: `w-[360px]` · `pt-[84px] pr-vintiga-xl pb-vintiga-xl` so the first rail card lines up with the page title.
- Mobile (<lg): rail is hidden — pages can drop the prop or render the rail content as a section in the body.

Style guide: new "Page Template" entry under Data Display with two examples — full layout (with tabs + rail) and a single-column form (no tabs, no rail). Vocabulary updated.

Affected: `ClubOverviewScreen` (auto-renew + accept-terms) and `ClubEditorEmailsScreen` (use-global-template).
- Style guide: new "Media" entry under Data Display with both populated + empty states. Vocabulary updated.
